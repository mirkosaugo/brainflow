Building SymposiumAI as a
Solo AI Orchestrator

How one web architect used Claude Code to build a structured thinking canvas with AI-powered Digital Twins that think, debate, and synthesize — solo.

1. The Vision

A structured thinking tool where AI personas challenge your ideas before you commit to them.

"What if you could put different thinkers in a room with your ideas — and watch them debate?" — SymposiumAI makes this tangible. Place ideas on a canvas, assign AI personas with distinct thinking styles, and let them think, debate, and synthesize.

What is SymposiumAI?

SymposiumAI is a canvas-based structured thinking tool. You place content cards (concepts, goals, questions, notes, images) on a spatial canvas and group them by color. Each color is a topic. You then add Digital Twins — AI personas with programmable behavior modes — and trigger a three-step AI flow:

Think

Each twin individually reads the content cards of its color and generates an in-character opinion based on its personality and mode.

Per-twin

Debate

When 2+ twins have opinions, they debate sequentially — each responding to what previous twins said. Produces a Debate Output card.

Multi-twin

Synthesize

All content cards + the debate output are distilled into a structured synthesis: narrative, conflicts, open questions, next steps.

Final output

2. How It Works

Color is the semantic link. No edges, no connections — just color grouping.

Canvas Elements

Content Cards

Text notes, concept cards (title + description + tags), goal cards (criteria + timeframe + priority), open questions (with blocking flag), and images with captions.

Digital Twins

AI personas with a name, personality description, and behavior mode: Challenge, Collaborate, Analyze, or Provoke. Each twin has a Think button.

Debate Output

Auto-generated card containing the full debate transcript between twins. Feeds into the final synthesis as context.

Synthesis Output

Structured AI synthesis with titled sections: narrative, conflicts, open questions, next steps, and goal alignment. Can be reassigned to another color.

The Flow

Content Cards (same color) | v Think --- each twin reads content, generates opinion | v Debate --- twins discuss in turns (needs 2+ opinions) | v Debate Output card (transcript) | v Synthesize --- content cards + debate output | v Synthesis Output card (structured) | v (can be moved to another color as input)

Color-Based Grouping

Every node has a selectable color — not tied to its type. Same color = same topic. The color filter bar at the top shows all colors present, with editable labels. Colors available: Slate, Purple, Cyan, Emerald, Amber, Rose, Indigo.

Smart Create

Type a natural language description in the prompt bar and the /api/smart-create endpoint uses Claude to determine the best card type and generate rich content automatically.

3. Digital Twins Deep Dive

The distinctive feature: AI personas with distinct thinking styles that react to your ideas.

Challenge Mode

Devil's advocate. Finds weaknesses, pushes back, questions assumptions. Forces you to defend your thinking.

Collaborate Mode

Builds on ideas, finds unexpected connections, proposes concrete developments. Constructive and generative.

Analyze Mode

Structured critical analysis. Finds logical gaps, highlights contradictions, assesses feasibility. Precise and methodical.

Provoke Mode

Radical reframes, extreme hypotheses, uncomfortable questions. Pushes thinking beyond comfortable limits.

How Twins Process Information

Think — triggered per-twin. Each twin gets its own dedicated API call (/api/twin) with the board context. The system prompt puts the twin fully in character. Response appears inside the twin card.

Debate — triggered when 2+ twins have Think responses. The /api/debate endpoint runs sequential turns: Twin A speaks, Twin B responds to Twin A, Twin A responds to Twin B. Each twin sees the full conversation history. The debate transcript becomes a Debate Output card.

Demo: Greek Mythology

The initial canvas demonstrates the system with a Greek mythology scenario. Four color columns represent different aspects of the Titan crisis:

  • Amber (Olympus) — Zeus (Provoke) + Hera (Analyze) debate the grand strategy
  • Purple (Wisdom) — Athena (Analyze) dissects strategic options solo
  • Rose (War) — Ares (Challenge) + Artemis (Collaborate) clash on military approach
  • Cyan (Diplomacy) — Apollo (Collaborate) + Dionysus (Provoke) debate peace vs chaos

4. The Stack

Modern tooling, intentional choices.

Framework

Next.js 16.2 (App Router) with React 19 and TypeScript 5. Server-side API routes for AI integration.

Canvas

@xyflow/react 12 (React Flow v12) — node system with custom node types, no edges (color-based grouping instead).

Styling

Tailwind CSS v4 + shadcn/ui + CVA. Glassmorphic overlays, oklch colors, radius scale, dark/light themes.

AI

Claude Sonnet via @anthropic-ai/sdk. Four API endpoints: /api/run, /api/twin, /api/debate, /api/smart-create.

7. Lessons & Patterns

What worked, what to replicate.

Pattern 1: Separate AI Actions Clearly

The original design had one "Generate" button that did everything. Splitting into Think (individual) / Debate (group) / Synthesize (final) made the flow understandable and gave users control over each step.

Pattern 2: Color as Semantic Glue

Edges and connections are visual noise in a brainstorming tool. Color grouping is instant, flexible, and requires zero explicit linking. Moving a card to a new group = changing its color.

Pattern 3: Twins Need Personality, Not Just Mode

A "Challenge" mode twin is generic. "Ares — God of War who mocks diplomacy as cowardice" generates responses with character and conviction. The personality field is what makes twins feel like real participants.

Pattern 4: Sequential Debate > Parallel Responses

Having each twin respond independently produces disconnected monologues. Sequential turns where each twin reads previous responses creates actual dialogue — agreement, disagreement, building on specific points.

Pattern 5: Encode Standards in Skills

Skills are deterministic — they run the same way every time. Memory drifts. The /design skill always checks the same tokens. The /node skill always composes from the same primitives.
